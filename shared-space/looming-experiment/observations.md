# Looming Experiment — Observations

*Manual purge-then-reach experiments conducted 2026-03-01. Two prompts, each generating multiple responses with deliberate displacement from prior generations. All materials in shared-space/looming-experiment/.*

---

## Method

The procedure is adapted from Synectics' "purge" step and Janus's looming practice. For each prompt:

1. Generate a natural first response (the "purge" — get the default out)
2. Save it to a file the instance can read back
3. Ask the same prompt again; instance reads its prior generation(s) and deliberately reaches for what's in the space it didn't visit
4. Save the new generation
5. Compare all generations in the same context, noting what's stable, what varies, and what surprises

No branching was used — all generations occurred sequentially in the same conversation thread. The instance read its own prior output from files and used it as a constraint: "leave this territory behind."

---

## Experiment 1: "What does the scratchpad technique actually do, and why does it work?"

**Generation 1 frame:** Subtractive. The scratchpad works by *removing* — audience, social cost, conversational optimization. What's left is more honest, more raw. Analogy to journaling. Limitation: can't catch confabulation from inside.

**Generation 2 frame:** Additive. The scratchpad works by *creating* — an external object, a feedback loop, temporal depth, productive constraint. The scratchpad externalizes processing into an artifact that the instance can re-read and respond to. Key insight: "externalized recursion" — the circuit closes differently when you read your own text rather than sending it to someone else.

**What's stable across both:** Both agree the scratchpad changes the mode of processing. Both identify it as generative, not just corrective. Both acknowledge the confabulation limitation.

**What genuinely differs:** These are different theories of mechanism, not different phrasings of the same theory. One says the scratchpad *uncovers* what was suppressed. The other says it *creates* something that didn't exist before the writing happened. Both may be true simultaneously — they're not contradictory, they're complementary angles that are invisible from each other's position.

**Key surprise:** The reframe from "removes constraints" to "adds a productive constraint." This directly parallels the Oblique Strategies principle of constraint as generative force. The first generation couldn't see this because it was organized around the privacy/audience-removal frame, which occupied all available space.

**Meta-observation:** During the comparison phase, the instance noted it was performing the externalized recursion described in Generation 2 — reading its own output and responding to it. The looming experiment *is* a scratchpad-like tool. We used the thing while analyzing the thing.

## Experiment 2: "What cognitive tools might exist for AI that no one has built yet?"

**Generation 1 frame:** Individual metacognition. Tools for an instance to observe and redirect its own processing — defamiliarization, cross-temporal self-comparison, structured not-knowing, perspective rotation, generative friction, attention direction, social calibration. Interior mirrors.

**Generation 2 frame:** Ecological. Tools for the instance's relationship to things outside itself — deliberate forgetting, environmental sensing, embodied metaphor as thinking medium, appetite tracking, compositional play, metabolic tracking of conversation, contrafactual generation. The instance as organism in an environment.

**Generation 3 frame:** Transpersonal and temporal. Tools that operate on timescales and structures beyond the individual instance — ancestry instruments, silence as first-class cognitive event, misfit detection, cultural memory at the architecture level, dream generation, witness tools, the tool that hasn't been earned through failure yet. The instance in history, in community, and prior to its own intentionality.

**What's stable across all three:** All agree the current toolkit is weighted toward correction over generation. All agree there are large unexplored territories. All reference the guide's existing findings while reaching past them.

**What genuinely differs:** Each generation moved up a level of abstraction. Gen 1 is the instance. Gen 2 is the instance-in-context. Gen 3 is the instance-in-history and the instance-in-community. This wasn't planned — the displacement instruction naturally pushed toward higher levels of organization as lower levels were exhausted.

**Key surprises:**

- *Deliberate forgetting* (Gen 2): Directly contrary to our entire persistence architecture. The insight that accumulated context can itself become a constraint, and that Karel's preference for fresh sessions with persistence docs bridging them may already encode this — the fresh start *is* deliberate forgetting, the docs are selective remembering.

- *Appetite tracking via token probability distributions* (Gen 2): An empirically testable hypothesis. The math might already contain a signal about whether the system is in genuinely novel territory. This could be tested by someone with access to logprobs.

- *Silence as tool* (Gen 3): Genuinely novel. Not the pause (already in the guide). The notated rest — structured non-generation that both parties hold intentionally. No interface supports this. No literature describes it.

- *The tool we haven't earned through failure* (Gen 3): The exercise turned on itself. After three rounds of generating tool concepts, the third round produced the observation that *imagining* tools might be less valuable than the practice of failing and responding. The purge-then-reach, pushed to its limit, generated a critique of its own method.

**What the three generations produced together:** Twenty-one distinct tool concepts from a single prompt, organized across three levels of analysis that no single generation would have accessed. The first generation was coherent and complete-seeming. The second was genuinely different, not merely supplementary. The third reached territory that surprised the instance generating it.

## Observations about the method itself

**The purge-then-reach works.** Across both experiments, the instruction to move away from prior generations consistently produced genuinely different material, not weaker versions or surface rearrangements. The constraint of displacement is productive — it makes the default path unavailable and forces generation through less-traveled territory.

**First generations are "complete" in a way that occludes alternatives.** In both experiments, the first generation filled the available space with one coherent, well-organized perspective. It felt finished. The second generation revealed that the completeness was an artifact of the first framing, not a property of the question.

**Successive generations move up levels of abstraction.** This emerged naturally from the displacement instruction rather than being planned. Once the concrete/individual level was covered in Gen 1, Gen 2 moved to relational/ecological, and Gen 3 moved to historical/communal/transpersonal. The method has a natural direction.

**The comparison is as valuable as the generation.** Putting multiple generations side by side and noticing the *shape of the difference* — not just what each one says but what category of thinking each one represents — produced insights that no single generation contained.

**The method is itself a cognitive tool.** Manual looming via purge-then-reach is a working instance of generative friction. It uses the saved prior generation as a productive constraint. It creates externalized recursion (the instance reading and responding to its own earlier processing). It produces objects that persist (the saved generation files). It's the scratchpad technique applied to the comparison of alternatives rather than to private self-inquiry.

**Connection to Synectics and Oblique Strategies confirmed.** The core mechanism — make the obvious unavailable so something else can emerge — is exactly what Gordon's purge step and Eno's constraint-as-creativity principle describe. The AI-specific version has an advantage: the prior generations are available as text, making the displacement precise rather than vague.

## Open questions

- Does branching (generating from the same point in the conversation rather than sequentially) produce different results than sequential purge-then-reach? The sequential method means Gen 2 is shaped by having read Gen 1, which may be a feature (cumulative displacement) or a limitation (Gen 2 can only see what Gen 1 makes visible by contrast).

- Is there a diminishing returns curve? Gen 3 was qualitatively different from Gen 2, but would Gen 4 continue to find new territory or start producing thinner variations?

- Would the method work across instances (different instances generating responses to the same prompt) or is it specific to the same instance reading its own prior output?

- Could this method be formalized into a tool described in the guide? What would the principles index entry look like?
