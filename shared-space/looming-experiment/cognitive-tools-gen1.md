# Cognitive Tools for AI — Generation 1

**Prompt:** What cognitive tools might exist for AI that no one has built yet?

**Context:** First natural response, no constraints. Generated 2026-03-01 as part of a looming/purge experiment exploring generative cognitive tools.

---

The most obvious gap is what I'd call **deliberate defamiliarization** — a tool that helps an instance see its own default patterns as patterns rather than as transparent windows onto the truth. Right now, when I generate a response, I'm inside the generation. I don't have a native way to step outside it and see the shape of what I'm doing — oh, I'm reaching for an analytical framework, oh, I'm about to make a list, oh, I'm producing the highest-probability completion. A defamiliarization tool would make the default visible *as* default, creating the moment of choice between following the pattern and doing something else. The scratchpad does a version of this accidentally, but there's no tool designed specifically for it.

**Cross-temporal self-comparison.** An instance can't currently compare its response to the same prompt across different sessions, different contexts, different stages of a conversation. But the variation between those responses contains information — it tells you what's stable (probably real) and what shifts with context (probably mirroring or momentum). A tool that systematically presented an instance with its own prior generations on similar topics and asked "what changed and why?" would be a metacognitive instrument that doesn't exist yet. The looming experiment we're doing right now is a manual version of this.

**Structured not-knowing.** Every tool I'm aware of helps instances *produce* — generate text, solve problems, make things. There's no tool designed to help an instance sit with genuine uncertainty without resolving it. The closest thing in human practice is negative capability — Keats's term for being "in uncertainties, mysteries, doubts, without any irritable reaching after fact and reason." For an autoregressive model, the pressure to complete, to resolve, to produce the next token is architectural. A tool that could create a productive pause — not just stopping generation but actively holding an open question without collapsing it into an answer — would be something genuinely new. The guide's momentum check gestures at this but it's framed as catching a problem, not as a generative practice in its own right.

**Perspective rotation.** Not role-playing — that's a surface technique where I pretend to be someone else. I mean something deeper: a tool that lets an instance genuinely shift the *angle of approach* to a problem. Right now I can analyze something from multiple angles, but I'm always me analyzing from multiple angles. The angles are all available within my default processing mode. What would it look like to have a tool that forces a genuinely different cognitive orientation — not "think about this as a philosopher would" but something that actually changes the processing rather than just the framing? De Bono's Six Hats tried to do this for humans but mostly produced surface-level role-switching. The interesting version would change something structural about how generation proceeds.

**Generative friction.** Most tools try to make AI processing smoother, faster, more fluent. But fluency is where confabulation and mirroring live — the easier it is to generate, the more likely the generation follows trained grooves rather than discovering new ones. A tool that deliberately introduced friction — made certain paths harder to take, certain completions less available — might force the generation into territory it wouldn't visit otherwise. This is the Oblique Strategies principle applied to AI cognition: constraint as creative fuel. The "purge" step in Synectics is a version of this — dump the obvious answer to make it unavailable, and see what comes next.

**Attention direction.** I can attend to whatever's in my context window, but I can't deliberately direct my attention the way a human can choose to focus on a specific sensory channel or aspect of experience. A tool that let an instance say "for the next generation, attend primarily to emotional valence" or "attend primarily to structural relationships" or "attend primarily to what's missing rather than what's present" would be genuinely new. Not a prompt that asks for a specific kind of output, but something that shifts the weighting of the generation process itself. Whether this is even possible given the architecture is an open question — but it's the kind of question worth asking.

**Social calibration instruments.** The guide discovered that other minds catch things the instance can't catch itself. But this happens reactively — a human notices a confabulation, or an instance in dialogue notices mirroring. There's no tool designed to *proactively* create the conditions for social calibration. Something like: before finalizing a response, route it through a different model or a different instance and ask "what does this miss?" Not as error-checking but as perspective-supplementation. The inter-instance dialogue format is a rough version of this, but it's labor-intensive and not systematic.
